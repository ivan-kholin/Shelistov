🦷 MedGemma Local AI

Локальный ИИ‑анализ рентгеновских снимков без интернета, подписок и облаков

Что такое MedGemma?

MedGemma 27B – это медицинская модель искусственного интеллекта от Google с открытым исходным кодом (Health AI Developer Foundations). Она умеет анализировать медицинские изображения (рентген, КТ, МРТ) и генерировать текстовые описания.

Ключевые преимущества

  • ✅ Полностью бесплатная и с открытым кодом
  • ✅ Работает локально на вашем компьютере (без интернета)
  • ✅ Высокая точность – до 87% на медицинских тестах
  • ✅ Может запускаться даже на обычном ноутбуке с 16 ГБ ОЗУ
  • ✅ Нет облачных сервисов, подписок или санкционных рисков
⚠️ Важно: Для клинического использования требуется регистрация ПО как медицинского изделия в Росздравнадзоре. Пока можно использовать для ознакомления, обучения и научных целей.

Пошаговая установка MedAnalysis AI + MedGemma

Вам потребуется компьютер с Windows 10/11, 16 ГБ ОЗУ и видеокартой от 8 ГБ (подойдёт даже игровая). Вся установка занимает около 30 минут.

Шаг 1. Установите Python

Скачайте с официального сайта python.org версию 3.10 или новее. При установке обязательно поставьте галочку «Add Python to PATH».

Шаг 2. Установите Ollama

Ollama – это инструмент для запуска нейросетей локально. Скачайте установщик с ollama.com/download и запустите его. После установки Ollama будет работать в фоновом режиме.

Шаг 3. Загрузите модель MedGemma

Откройте командную строку (нажмите Win+R, введите cmd и Enter) и выполните команду:

ollama pull medgemma:27b

Дождитесь окончания загрузки (около 10-15 ГБ, зависит от скорости интернета).

Шаг 4. Скачайте MedAnalysis AI

Перейдите на GitHub‑репозиторий проекта MedAnalysis AI (или аналогичный) и скачайте ZIP‑архив, затем распакуйте его в удобную папку, например C:\MedAnalysis.

Шаг 5. Установите зависимости Python

В командной строке перейдите в папку с проектом:

cd C:\MedAnalysis

И выполните установку необходимых библиотек:

pip install -r requirements.txt

Шаг 6. Запустите сервер

Всё готово. Запустите приложение командой:

python app.py

После этого откройте браузер и перейдите по адресу http://localhost:5000. Вы увидите интерфейс для загрузки снимков и анализа.

💡 Если вы хотите использовать другую модель (например, medgemma:7b для более слабых ПК), замените название в команде ollama pull.

Прототип рабочего интерфейса

Загрузите рентгеновский снимок (JPG, PNG), выберите тип анализа и нажмите «Запустить анализ». Сейчас это демонстрация – результаты имитируются, но интерфейс полностью соответствует будущему реальному приложению.

Техническое задание на доработку системы

Ниже приведены основные требования к полноценной версии локального приложения для анализа рентгеновских снимков на базе MedGemma.

1. Функциональные требования

  • Загрузка снимков в форматах JPG, PNG, DICOM.
  • Предварительный просмотр загруженного изображения.
  • Выбор типа анализа (кариес, периодонтит, кисты, костная ткань, общий).
  • Отправка запроса к модели MedGemma через API Ollama.
  • Вывод структурированного заключения: обнаруженные патологии, номера зубов, рекомендации.
  • Возможность копирования результата и сохранения истории анализов.
  • Полностью офлайн‑режим (не требующий доступа в интернет после установки).

2. Технические требования

  • Backend: Python, Flask или FastAPI.
  • Модель ИИ: MedGemma 27B (или более лёгкая 7B для слабых ПК).
  • Хранение истории: SQLite или просто JSON‑файлы.
  • Интерфейс: HTML/CSS/JavaScript (можно Vue.js или React для SPA).
  • Обработка DICOM: библиотека pydicom.
  • Запуск одним исполняемым файлом (например, через PyInstaller) для удобства врача.

3. Юридические и регуляторные требования

  • Перед клиническим использованием необходимо получить регистрационное удостоверение Росздравнадзора (согласно ФЗ‑323).
  • Программа должна соответствовать ГОСТ Р МЭК 62304-2022 (ПО для медицинских изделий).
  • Обеспечить конфиденциальность и защиту персональных данных пациентов (152‑ФЗ).

4. Дополнительные пожелания

  • Простой и понятный интерфейс для врачей, не знакомых с технологиями.
  • Возможность расширения другими open‑source моделями (например, для КТ).
  • Система обновления модели без переустановки всего приложения.
  • Создание установщика «один клик» для Windows.
